1. What Is a Process?


1.1 Program vs. Process
Program: Code stored in memory waiting to be run — a static entity on disk.
Process: The representation of a running program — a dynamic, active entity. (Prof: opening Chrome = 1 process; opening Chrome again = another process)


1.2 Why Processes Matter
The hardware recognizes instructions, not 'programs running' — but the process abstraction is convenient for us
Multiple processes share the CPU via scheduling — from the OS's perspective, each process takes turns on the CPU
The OS must interleave execution of multiple processes to maximize CPU utilization
The OS must allocate resources to processes per policy, avoiding deadlock

1.3 Process Control Block (PCB)
The PCB is a table stored in memory containing EVERYTHING the process needs. It is the most important data structure in the OS. Without it, a process cannot be managed, interrupted, or resumed.



1.4 Trace
A trace = the sequence of instructions executed by a process
The OS interleaves the traces of multiple processes on the CPU
A dispatcher program handles switching between processes

2.3 Process Suspension (Swapping)
Swapping is NOT the same as switching! It is the process of moving a process image to secondary storage (disk) to free up RAM.


Why suspend? A web server listening on port 80 is idle most of the time — instead of letting it reserve memory, swap it to disk so other processes can use that space.

Termination Events:
Normal completion (process calls OS 'exit')
Time limit exceeded, memory unavailable, bounds violation
Arithmetic error, I/O failure, invalid instruction
Parent terminates child, or parent itself terminates

4.2 Process Switching
A process switch (context switch) replaces the running process on the CPU with another. This is more expensive than a mode switch.


What triggers a process switch?
Clock interrupt: Time slice expired — current process moved to Ready
I/O interrupt: I/O completed — blocked processes moved to Ready
Trap: Error or exception in running process
Supervisor call: User process requests OS service (e.g., file open)

6. Process Creation with fork() (C Code)


The UNIX fork() system call creates a child process that is an exact copy of the parent.

When fork() is called, a child process is created. Both processes continue executing from the line after fork(). The key is what each process gets as the return value of fork().
